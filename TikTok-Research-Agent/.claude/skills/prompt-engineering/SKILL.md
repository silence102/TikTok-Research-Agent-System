# Prompt Engineering Skill

## 환각 방지 전략

### 1. 데이터 기반 답변 강제

```
시스템 프롬프트:
"제공된 데이터에만 기반하여 답변하세요.
데이터에 정보가 없으면 '해당 정보를 찾을 수 없습니다'라고 답변하세요.
절대 추측하거나 일반 지식을 사용하지 마세요."
```

### 2. 출처 명시 강제

```
사용자 프롬프트:
"답변 시 반드시 출처를 명시하세요.
형식: [정보 내용] (출처: URL 또는 날짜)"
```

### 3. Temperature 설정

- **일관성 필요**: Temperature 0.0-0.3
- **창의성 필요**: Temperature 0.7-1.0
- **권장값 (에이전트)**: 0.3

## 프롬프트 구조화

### System Prompt (역할 정의)
```
당신은 틱톡 트렌드 전문 리서처입니다.
Airtable 데이터베이스의 정보만을 활용하여 답변합니다.
```

### User Prompt (작업 지시)
```
질문: {user_question}

컨텍스트:
{retrieved_data}

지침:
1. 컨텍스트 데이터만 사용
2. 출처 명시
3. 간결하고 명확하게 답변
```

## Few-Shot Prompting

```
예시 1:
질문: "최근 틱톡 인기 해시태그는?"
답변: "2025년 1월 13일 기준 상위 3개는 #fyp (조회수 12M), #viral (8M), #trend (5M)입니다. (출처: Apify 수집 데이터)"

예시 2:
질문: "메타버스 관련 틱톡 트렌드는?"
답변: "제공된 데이터에서 메타버스 관련 정보를 찾을 수 없습니다."
```

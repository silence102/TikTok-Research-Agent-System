## 🚀 Phase 2: 실전 프로젝트 (TikTok Research Agent System)

### 프로젝트 소개

**이름**: TikTok Research Agent System
**목적**: 틱톡 트렌드 및 크리에이터 이코노미 리서치 자동화

**문제 정의**:
- 마케터가 틱톡 트렌드 파악에 매일 2시간 소요
- 크리에이터 이코노미 뉴스 수동 수집 및 요약
- 데이터 기반 의사결정 어려움

**솔루션**:
3개의 AI 에이전트가 자동으로 데이터 수집 → 요약 → 질의응답

```
[에이전트 1] 틱톡 트렌드 자동 수집 (매일 09:00)
    ↓
[Airtable 저장]
    ↓
[에이전트 2] 크리에이터 이코노미 뉴스 수집 (매일 10:00)
    ↓
[Airtable 저장]
    ↓
[에이전트 3] 사용자 질문 → 데이터 기반 답변 (언제든지)
```

### 문서 구조 및 목적 분류

프로젝트 문서를 **5가지 핵심 질문**에 따라 분류하여 체계적으로 구성했습니다:

#### 📄 docs/Ideation.md - "왜?"

**목적**: 프로젝트 탄생 배경과 아이디어 발상 과정

**내용**:
- 문제 인식 (채용공고 분석 → 도메인 선택)
- 프로젝트 목적 (Primary/Secondary Goals)
- 핵심 아이디어 (3가지 에이전트 시스템)
- 차별화 포인트 (기존 접근법 vs 우리 방식)
- 창의적 요소 (맥도날드 시스템 적용, 7단계 워크플로우)
- 기대 효과 및 성공 기준

**분류 기준**: 의도와 동기를 명확히 기록 (바이브코딩 원칙 - 인간의 책임)

---

#### 📄 docs/PRD.md - "무엇을?"

**목적**: 제품 요구사항 정의서 (Product Requirements Document)

**내용**:
- Goal / Non-Goal 명시
- 우선순위 분류 (Must-have / Should-have / Nice-to-have)
- 사용자 여정 (User Journey)
- 기능 명세서 (에이전트 1, 2, 3 상세 스펙)
- UX 설계 (CLI 인터페이스)
- 성과 지표 (KPI)
- 제약 조건 및 릴리즈 계획

**분류 기준**: "기획서만 잘 쓰면, 뒷단은 마법같이 풀린다" (바이브코딩 핵심)

---

#### 📄 docs/Tasks.md - "언제?"

**목적**: 1주일 MVP 완성을 위한 Day-by-Day 작업 계획

**내용**:
- 전체 일정 개요 (Day 1-7)
- Day별 상세 체크리스트:
  - Day 1-2: 환경 구축 + 에이전트 1
  - Day 3-4: 에이전트 2
  - Day 5-7: 에이전트 3 + 문서화
- 마일스톤 (Milestone 1-3)
- 리스크 관리 (시간 부족, API 비용, 기술적 막힘)
- 최종 체크리스트

**분류 기준**: 실행 계획의 시간축 관리 (바이브코딩 - 빠른 실행력)

---

#### 📄 docs/TechStack.md - "어떻게?"

**목적**: 기술 선택과 아키텍처 설명

**내용**:
- 전체 기술 스택 개요 (계층별 구조)
- 핵심 기술:
  1. n8n (워크플로우 자동화)
  2. Claude 3.5 Sonnet (고품질 LLM)
  3. GPT-4o mini (저비용 LLM)
  4. Airtable (구조화 데이터베이스)
  5. ChromaDB (벡터 DB, Should-have)
  6. Apify (TikTok 데이터 수집)
  7. RSS Feeds (뉴스 수집)
  8. Python 3.11 + Rich (CLI)
- 시스템 아키텍처 (데이터 흐름도)
- 기술 선택 의사결정 (대안 비교)
- 성능 및 제약 사항

**분류 기준**: 기술적 구현 방법론 (바이브코딩 - 도메인별 문서 분리)

---

#### 📄 docs/Tutorial.md - "따라하기"

**목적**: 처음 시작하는 사람을 위한 단계별 가이드

**내용**:
- 사전 준비 (필수 소프트웨어, 계정 생성)
- Step 1-10 상세 튜토리얼:
  1. 프로젝트 클론 및 설정
  2. API 키 설정
  3. Airtable 테이블 생성
  4. n8n 설정 및 에이전트 1 실행
  5. 에이전트 2 실행
  6. 에이전트 3 CLI 실행
  7. CLAUDE.md 활용법
  8. 데모 시나리오 실행
  9. 최종 점검
  10. 다음 단계
- 트러블슈팅

**분류 기준**: 실무 사용성 (비기술자도 따라할 수 있는 가이드)

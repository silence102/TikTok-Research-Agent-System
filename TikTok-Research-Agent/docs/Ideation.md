# 아이디어 발상 및 프로젝트 목적

## 📌 프로젝트 탄생 배경

### 문제 인식
삼양식품 DXT팀 채용공고를 분석하면서 다음과 같은 핵심 요구사항을 발견했습니다:

1. **"최근 1개월 내 본인이 직접 만든 에이전트 2개 이상"**
2. **n8n 워크플로우 자동화 도구에 대한 깊은 이해**
3. **다양한 LLM API 활용 경험**
4. **프롬프트 엔지니어링 능력**
5. **실험 우선 문화: "작동하는 에이전트를 먼저 만들고 개선하는" 방식 선호**

### 도메인 선택 이유

**"틱톡 트렌드 및 크리에이터 이코노미"**를 선택한 3가지 이유:

1. **팀의 핵심 도메인**
   - DXT팀의 주력 프로덕트 **Popow**는 마이크로/나노 틱톡커(1K-50K 팔로워)와 브랜드를 연결
   - 채용공고에서 명시된 주요 업무: "틱톡 트렌드 분석", "바이럴 영상 분석", "크리에이터 온보딩 심사"

2. **실무 적용 가능성**
   - 틱톡 시장은 빠르게 변화하며, 매일 새로운 트렌드가 생성됨
   - 마케터/PM이 수동으로 리서치하는 데 하루 2시간 이상 소요
   - **자동화의 ROI가 명확함**: 시간 90% 단축 가능

3. **기술 스택 증명 기회**
   - 데이터 수집 (Apify, RSS)
   - LLM API 활용 (Claude, GPT)
   - 워크플로우 자동화 (n8n)
   - RAG 구현 (질의응답)

## 🎯 프로젝트 목적

### Primary Goal (핵심 목표)

**"실행력 있는 AI 빌더" 역량을 증명하는 포트폴리오 완성**

- 채용 필수 요건 모두 충족
- 바이브코딩 원칙 실천 (200줄 CLAUDE.md + 작동하는 프로토타입)
- 1주일 내 MVP 완성

### Secondary Goal (부차적 목표)

1. **실무 활용 가능성 증명**
   - 비즈니스 임팩트가 명확한 에이전트 (리서치 시간 90% 단축)
   - 비기술자도 사용 가능한 인터페이스 (CLI)

2. **기술 역량 다각도 증명**
   - n8n: 3개 워크플로우 실전 작동
   - LLM API: Claude + GPT 혼용 전략
   - 프롬프트 엔지니어링: 환각 방지, RAG

3. **학습 및 성장**
   - n8n 마스터리
   - LLM API 최적화 (토큰 비용 관리)
   - 바이브코딩 방법론 체화

## 💡 핵심 아이디어

### 3가지 에이전트 시스템

**아이디어의 핵심**: "자동 수집 + 자동 요약 + 질의응답"의 3단계 파이프라인

```
[에이전트 1] 틱톡 트렌드 자동 수집 (매일 09:00)
    ↓
[데이터 저장소] Airtable
    ↓
[에이전트 2] 크리에이터 이코노미 뉴스 수집 (매일 10:00)
    ↓
[데이터 저장소] Airtable
    ↓
[에이전트 3] 사용자 질문 → 데이터 기반 답변 (언제든지)
```

### 차별화 포인트

1. **3개 독립 에이전트 = 모듈화**
   - 각 에이전트가 독립적으로 작동
   - 하나가 실패해도 나머지는 정상 작동
   - 유지보수 용이

2. **데이터 중심 설계**
   - Airtable을 중앙 데이터베이스로 활용
   - 구조화된 데이터 → 환각 방지
   - 출처 추적 가능

3. **실용성 우선**
   - 복잡한 ML 모델 대신 LLM API 활용
   - 프로덕션 레벨 확장성보다 포트폴리오 증명 집중
   - MVP 1주일 완성 가능

## 🔍 경쟁 분석 (기존 접근법 vs 우리 방식)

### 기존 접근법의 문제점

| 접근법 | 문제점 | 비용/시간 |
|--------|--------|-----------|
| **수동 리서치** | 마케터가 매일 틱톡 앱 + 구글링으로 수집 | 2시간/일 |
| **틱톡 공식 API** | 엔터프라이즈 전용, 고비용 | $500+/월 |
| **단일 크롤러** | 데이터만 수집, 인사이트 없음 | 개발 2주 |
| **ChatGPT 직접 사용** | 최신 데이터 없음, 환각 위험 | - |

### 우리 방식의 장점

| 특징 | 장점 | 비용/시간 |
|------|------|-----------|
| **3개 에이전트 시스템** | 자동 수집 + 요약 + 질의응답 통합 | 1주일 개발 |
| **Apify + RSS** | 저비용 데이터 수집 | $5/월 |
| **LLM API 혼용** | Claude(고품질) + GPT(저비용) 최적화 | $3/월 |
| **RAG 구현** | 데이터 기반 답변 → 환각 방지 | - |
| **n8n 자동화** | 크론 스케줄링으로 무인 작동 | - |

**결과**: 월 $8 이하 비용으로 하루 2시간 → 10분 (90% 시간 단축)

## 🎨 창의적 요소

### 1. "맥도날드 시스템" 적용

바이브코딩 가이드에서 배운 "맥도날드 프랜차이즈 모델"을 에이전트 개발에 적용:

- **테크리드 (본사)**: CLAUDE.md 200줄 핵심 가이드 작성
- **바이브코더 (지점장)**: 의도와 의사결정 기록
- **AI 에이전트 (알바)**: 정해진 매뉴얼대로 실행

### 2. "7단계 워크플로우" 준수

바이브코딩의 7단계를 철저히 따름으로써 체계적 개발:

1. ✅ 컨텍스트 확보: 채용공고 + 바이브코딩 가이드 분석
2. ✅ 요구사항 정의: PRD 작성 (Goal/Non-Goal/Must-have)
3. ✅ UX 설계: CLI 인터페이스 설계
4. ✅ 의존성 분석: Apify, Claude API, Airtable
5. ✅ 구현 계획: 3개 에이전트 순차 개발
6. ✅ 리스크 분석: API 한도, 환각, 비용
7. ✅ 테스트 작성: 샘플 질문 20개 검증

### 3. "휘발성 컨텍스트" 개념 활용

- 중요한 의사결정만 PRD, PROMPTS.md에 기록
- 임시 실험은 로컬에서만 (Git에 커밋 안 함)
- CLAUDE.md는 200줄 엄수 → 토큰 절약

## 📝 핵심 인사이트

### 배운 점 (예상)

1. **n8n의 강력함**
   - 비개발자도 워크플로우 구축 가능
   - 다양한 통합 (Apify, Claude, Airtable)
   - 크론 스케줄링 무인 작동

2. **LLM API 최적화 전략**
   - Claude: 고품질 요약/답변 (비용 높음)
   - GPT-4o mini: 간단한 파싱 (비용 낮음)
   - 캐싱으로 토큰 절약

3. **프롬프트 엔지니어링의 중요성**
   - "데이터에 없으면 '정보 없음'"으로 환각 방지
   - Temperature 0.3으로 일관성 확보
   - 출처 강제로 신뢰성 향상

### 전략적 의사결정

**왜 이 기능은 포함하고, 저 기능은 제외했는가?**

| 기능 | 결정 | 이유 |
|------|------|------|
| **3개 에이전트** | ✅ 포함 | 채용 요건 "2개 이상" 충족 + 완전한 시스템 |
| **CLI 인터페이스** | ✅ 포함 (MVP) | 빠른 구현, 데모 용이 |
| **Streamlit UI** | ❌ Should-have | MVP 범위 초과, 1주일 내 불가 |
| **ChromaDB 벡터 검색** | ❌ Should-have | 심플 RAG로 충분, 복잡도 증가 |
| **트렌드 예측 모델** | ❌ Nice-to-have | ML 모델 개발은 과한 투자 |
| **Slack 통합** | ❌ Should-have | n8n에 Slack 노드만 추가하면 되지만 우선순위 낮음 |

**핵심 원칙**: "작동하는 것을 먼저 만든다" (바이브코딩)
